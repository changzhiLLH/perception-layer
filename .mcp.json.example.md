# MCP 配置文件

将这个文件重命名为 `.mcp.json` 放在你的项目根目录，Claude Code 启动时会自动发现并提示加载。

## 配置内容

```json
{
  "mcpServers": {
    "perception-layer": {
      "command": "perception-layer"
    }
  }
}
```

## 说明

| 字段 | 含义 |
|------|------|
| `command` | `perception-layer` 是 pip install 安装的命令。如果 `perception-layer` 不在 PATH，改用 `python -m perception_layer` |
| **工作目录** | MCP server 默认监听 Claude Code 启动时的工作目录（即你的项目目录）。可通过命令行参数指定其他目录：`perception-layer /path/to/watch` |

## 自定义忽略路径

配置文件在 `~/.perception-layer/config/sensor_ignore.json`（pip install 模式）或项目根 `config/sensor_ignore.json`（源码运行模式）。

编辑 `sensor_ignore.json` 添加不想监听的目录：

```json
{
  "version": 1,
  "ignore_paths": [
    "node_modules/",
    ".git/",
    "dist/",
    "build/"
  ]
}
```

## 自定义路由规则

配置文件在相同位置的 `routing_rules.json`。路由规则决定哪些事件落盘（可持久查询），哪些只保留在内存。

```json
{
  "version": 1,
  "rules": [
    {"prefix": "/tmp/", "action": "ring_only", "reason": "临时文件不落盘"},
    {"prefix": "/", "action": "persist", "reason": "默认落盘"}
  ]
}
```

- `persist`：事件写入文件，可通过 `query_by_handle` 回查
- `ring_only`：事件只在内存保留，不落盘
